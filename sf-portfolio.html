Project 02
Multi-Org Data Sync Framework
Real-time, conflict-aware record synchronisation across Salesforce orgs — zero middleware
View on GitHub →
The Business Challenge

Enterprises that grow through acquisitions or operate global-regional Salesforce architectures constantly wrestle with a painful question: how do you keep customer, product, or operational data in sync across multiple Salesforce orgs without expensive middleware platforms like MuleSoft or Informatica?

Without a sync framework, teams maintain duplicate data manually, integrations break silently, and compliance teams struggle to maintain a single source of truth across entities.

The Innovation

This framework uses Salesforce Platform Events as a native message bus between orgs — no external tools, no per-message cost, no infrastructure to manage. It includes built-in conflict detection (last-write-wins with local modification check) and automatic retry with exponential backoff.

Business Value

Organisations that previously paid for middleware licences can replace point-to-point integrations with a governed, auditable sync layer that lives entirely inside Salesforce — reducing integration cost, improving data accuracy, and cutting sync latency from batch-hours to real-time seconds.

Project 03
Case Dependency & Blocking Chain Framework
Service Cloud — the only solution that enforces case interdependencies natively in Salesforce
View on GitHub →
The Business Challenge

In complex service environments — IT, telecom, manufacturing — one customer issue often blocks resolution of many others. A network outage creates 50 customer cases, all of which are stuck until the root cause is fixed. Salesforce has no native concept of case dependencies.

Service agents waste time attempting to resolve cases they cannot close. Customers receive inaccurate status updates. Supervisors have no visibility into how many cases are blocked behind a single issue. SLA breach rates climb.

The Innovation

This solution introduces a many-to-many dependency model between Cases. When a "blocker" case is closed, the system automatically unblocks all downstream cases and notifies the respective agents by email — no manual coordination required. An embedded LWC shows the full dependency chain visually on every Case record.

Business Value

Service teams eliminate wasted effort on unresolvable cases. SLA compliance improves because agents focus only on cases they can actually close. Supervisors gain real-time visibility into systemic issues and their downstream blast radius.

Project 04
Dealer Audit & Service Level Framework
End-to-end configurable dealership audit system with automated service level management and PDF reporting
View on GitHub →
The Business Challenge

Manufacturers with dealer or franchise networks need to regularly audit dealerships and tie the results to commercial consequences — specifically, how much of the service cost the manufacturer will absorb. Managing this with spreadsheets, emails, and manual scoring creates inconsistency, disputes, and delayed outcomes.

There is no standard Salesforce product that handles configurable audit questionnaires, weighted scoring, automatic service level promotion and demotion, or formal PDF result communication — all in one place.

The Innovation

A fully admin-configurable system — every service level, scoring threshold, question, and weightage is set by the business without code changes. Auditors use a mobile-friendly tabbed form with photo upload. The system tracks consecutive audit performance to promote or demote dealerships automatically after N consecutive passing or failing audits.

Business Value

Manufacturers eliminate disputes through documented, weighted, evidence-backed audit results delivered as professional PDF reports. Dealerships receive transparent scoring with a structured re-audit appeal process. Service cost allocation is automated and traceable.

Project 05
SF AI Insights Reporter
Ask a plain-English question, receive a rich AI-generated business report by email — entirely inside Salesforce
View on GitHub →
The Business Challenge

Business users need instant answers from Salesforce data but can't write SOQL and can't wait for IT to build reports. Standard reporting tools require pre-defined queries, rigid schemas, and BI expertise — none of which a sales manager or service director has time for.

The only existing alternatives are expensive BI platforms with their own licensing, maintenance overhead, and data sync delays.

The Innovation

This solution makes Claude AI a live Salesforce data analyst. The user types a plain-English question, Claude generates and self-heals the SOQL, queries live data, then produces a full HTML report (charts, metric cards, data table) and emails it as an attachment — all within Salesforce, zero external infrastructure.

A self-learning knowledge base records every successful and failed query pattern, making the system progressively more accurate with each report run.

Project 06
AI Code Analyzer
AI-powered Salesforce APEX & LWC code reviewer with configurable guardrail rules, PMD static analysis, and streaming findings
View on GitHub →
The Business Challenge

Salesforce teams accumulate technical debt faster than it can be reviewed. Manual code review is inconsistent — different reviewers catch different issues, guardrail standards get applied unevenly, and junior developers don't always know what "best practice" looks like for Apex bulkification, FLS checks, or LWC reactivity.

PMD catches syntactic violations but misses semantic issues that require understanding of architecture, naming conventions, and intent.

The Innovation

A two-pass review engine: PMD runs deterministic static rules first, then Claude performs deep semantic review guided by a library of 70+ configurable guardrail rules organised by category (security, performance, architecture, naming, testing, governance, accessibility). Rules are authored in plain Markdown — any team member can add or modify them without touching code.

Findings stream in real-time via Server-Sent Events. Each finding links to the exact code line with severity highlighting, rationale, and a concrete fix suggestion.

Business Value

Development teams get consistent, comprehensive code reviews in seconds rather than days. Senior engineers spend time mentoring rather than hunting for SOQL-in-loops violations. Guardrail rules encode institutional knowledge that persists beyond individual contributors.
